# P1 H5 Funds Cockpit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a mobile-first “B · 资金指挥舱” H5 that completes the available-cash, card, upload, planning, alert, and report workflow under `/szt/`.

**Architecture:** A React Router application owns page composition, TanStack Query owns server state, and a typed `apiClient` owns same-origin cookie/CSRF transport. Feature folders expose focused pages and components; shared design tokens keep mobile bottom navigation and desktop sidebar visually consistent.

**Tech Stack:** React 19, TypeScript 5, Vite 7, React Router 7, TanStack Query 5, Recharts 3, Lucide React, Vitest, Testing Library, Playwright.

## Global Constraints

- Source root is `frontend/h5/`; do not modify `frontend/miniprogram/`.
- Vite `base` is exactly `/szt/`.
- Use relative API URLs under `/api/v1/`; never hard-code a development host in page code.
- Use the approved deep-ink “B · 资金指挥舱” visual system.
- Mobile navigation has exactly five entries: 首页、资金、规划、提醒、我的.
- Desktop uses the same routes with a left sidebar.
- Available cash is optional; null always displays “试算”.
- Email and SFTP are disabled cards labeled “正在开发中”.
- No page may expose tokens, stack traces, SQL, or internal server paths.
- Every API-backed page implements loading, empty, error, and normal states.

---

## File Structure

- `frontend/h5/src/app/`: router, providers, application shell.
- `frontend/h5/src/api/`: typed contracts, client, session bootstrap, query keys.
- `frontend/h5/src/components/`: shared UI primitives and state panels.
- `frontend/h5/src/features/dashboard/`: funds cockpit and 30-day timeline.
- `frontend/h5/src/features/funds/`: available cash, cards, upload, disabled connectors.
- `frontend/h5/src/features/planning/`: recommendation, schedule, stop-loss.
- `frontend/h5/src/features/alerts/`: repayment and gap alerts.
- `frontend/h5/src/features/report/`: monthly health report and data-quality labels.
- `frontend/h5/e2e/`: browser-level core flow.

- [ ] **Step 3: Implement same-origin client and providers**

`apiClient` always sets `credentials: "include"`, `Accept: "application/json"`, sets JSON content type only for non-FormData bodies, and reads `szt_csrf` for unsafe methods. On 401 it calls `/api/v1/auth/session` once and retries once; a second 401 becomes an `ApiError`.

`AppProviders` creates one `QueryClient` with one retry for network failures and no retry for 4xx responses. `AsyncState` presents concise Chinese copy and a visible retry button while preserving page layout.
