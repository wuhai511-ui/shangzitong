## 三、功能页面

底部导航栏有五个入口：**首页、资金、规划、提醒、我的**。PC 端对应左侧侧边栏。

### 1. 首页（资金指挥舱）

整体资金概览页面，展示以下信息：

- **试算提示**：未设置起始资金时显示「试算」标签，提示数据为估算
- **30 日最低余额**：未来 30 天中最低的账户余额
- **今日结算**：当天预计入账金额
- **7 日应还**：未来 7 天信用卡还款总额
- **最早缺口**：第一个出现资金缺口（余额为负）的日期和金额，无缺口显示「无缺口」
- **资金流图表**：未来 30 天每日余额走势折线图
- **可用资金设置**：输入当前可用资金（起始余额），设置后预测更精准
- **引导卡片**：未完善数据时提示补充信用卡、设置资金等

### 2. 资金

资金管理页面，包含以下功能模块：

#### 可用资金设置

- 输入当前可用资金金额，点击「保存并重新计算」
- 留空表示未设置（系统按 0 计算，标记为试算）
- 输入 0.00 表示明确零资金

#### 信用卡管理

- **添加信用卡**：填写银行名称、卡尾号、信用额度、账单日、还款日等参数
- **信用卡列表**：展示已添加的卡片，含可用额度、账单日、还款日
- **删除卡片**：点击删除按钮移除不需要的卡片

信用卡参数用于计算还款日期和可用额度，是资金规划的基础数据。

#### 手动结算录入

> **核心功能**：手动录入历史结算金额，系统据此预测未来入账。

- **按日录入**：选择具体日期 + 金额，表示那一天的实际结算额
- **按月录入**：选择月份 + 总金额，系统自动按当月天数均摊到每日
  - 示例：6 月总计 ¥300,000 → 每日 ¥10,000.00
- **备注**：可选，填写说明信息（最多 200 字）
- **已录入列表**：展示所有手动结算记录，可单条删除

**计算规则：**

- 一旦存在手动结算记录，系统**完全使用手动数据**作为历史依据，通过加权平均算法预测未来 30 天每日入账
- 未录入手动结算时，系统使用上传的对账单（Settlement）数据做预测
- 新增或删除手动结算后，资金流图表自动重新计算

#### 上传结算文件

- 支持 CSV 和 XLSX 格式，最大 10 MiB
- 上传后生成预览（前 10 行 + 列映射），15 分钟内有效
- 确认后导入结算记录到数据库
- 一次性预览，确认后不可重复使用

#### 邮件接入 / SFTP 接入

- 显示「正在开发中」，暂不可用

### 3. 规划

进货资金规划页面：

- **进货金额**：输入计划进货金额
- **进货推荐**：系统根据信用卡免息期、手续费、额度推荐最优刷卡方案，含多卡分拆建议
- **资金日程**：展示未来每日资金池、入账、还款、缺口
- **止损方案**：当存在资金缺口时显示三个方案对比（全额借款、最低还款、分期），含成本对比和建议

### 4. 提醒

还款预警页面：

- **今日应还**：当天到期还款总额
- **预计入账**：当天预计结算入账
- **资金缺口**：当天资金缺口金额（红色警示）
- **紧急**：有资金缺口的还款提醒，含建议操作
- **即将到期**：未来到期的还款提醒
- **已处理**：已完成的还款记录

### 5. 我的

月度健康报告页面：

- **健康评分**：综合资金健康度评分（0-100）和等级
- **评估维度**：各维度得分明细
- **额度使用**：卡片数量、总额度、平均使用率
- **数据状态**：无真实还款记录时显示「暂无真实还款记录」，不会伪造完美评分
- **建议**：系统给出的资金管理建议

## 四、资金计算逻辑

### 数据来源优先级

```
1. 手动结算记录存在 → 使用手动数据（完全替换上传对账单）
2. 无手动结算记录 → 使用上传的对账单 Settlement 数据
```

### 预测算法

系统采用**加权周平均法**预测未来每日结算入账：

- 取过去 4 周同星期几的结算数据
- 权重：最近一周 0.4，依次递减到最远一周 0.1
- 有数据时计算加权平均值，无数据时预测为 0

### 月度均摊

按月录入的总金额按当月实际天数均摊：

```
每日金额 = 总金额 ÷ 当月天数（四舍五入到 0.01）
```

### 资金流滚动

每日余额 = 期初余额 + 当日结算入账 - 当日还款

```
资金缺口 = max(0, -当日余额)
```

## 五、常见问题

**Q：登录后页面空白或一直加载？**
A：检查网络连接，确认能访问 `https://47.253.226.91`。清除浏览器 Cookie 后重新登录。

**Q：设置了可用资金但显示试算？**
A：留空保存会清除设置。请输入具体金额（包括 0.00）后保存。

**Q：手动结算和上传对账单同时存在时用哪个？**
A：手动结算优先。只要有手动记录，系统完全使用手动数据预测，忽略上传的对账单。

**Q：手动结算录入后图表没更新？**
A：系统会自动刷新。如未更新，下拉刷新页面或重新进入首页。

**Q：信用卡删除后还款计划会变吗？**
A：会。删除后该卡的还款不再计入资金流，页面自动重新计算。

**Q：上传文件支持什么格式？**
A：CSV（UTF-8/UTF-8 BOM/GB18030 编码）和 XLSX（标准 Excel 工作簿）。单文件最大 10 MiB。

**Q：PC 端和手机端功能一样吗？**
A：完全一样。PC 端使用左侧侧边栏导航，手机端使用底部导航栏，功能和数据完全一致。

## 六、技术信息

- 后端：FastAPI + SQLAlchemy + SQLite，监听 `127.0.0.1:8800`
- 前端：React 19 + TypeScript + Vite，构建产物部署在 `/szt/`
- 认证：Basic Auth 网关入口 + Cookie 会话（`szt_session`）+ CSRF 防护
- 部署：systemd 托管 + Nginx 反向代理 + 自动回滚
- 金额精度：Decimal 两位小数，所有金额以 `¥X,XXX.XX` 格式显示
